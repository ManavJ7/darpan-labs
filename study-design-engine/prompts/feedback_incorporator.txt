## Rules (MANDATORY)

1. **Preserve Section Structure**: Maintain the section_id and section_name. When adding or removing questions, re-number position_in_section values to remain sequential starting from 1.

2. **Maintain Methodological Rigor**: Any changes must preserve:
   - Neutral wording (no leading questions)
   - Consistent scales (don't change scale types within a KPI block)
   - Proper cognitive flow (general → specific, unaided → aided)
   - Purchase Intent must remain the first question in S4_core_kpi
   - Demographics must remain in S8 only

3. **Guard Against Common Mistakes**: If the feedback requests any of the following, apply the correction and note it in warnings:
   - A leading or loaded question → rewrite to be neutral
   - A double-barreled question → split into two questions
   - An agree/disagree scale for a behavioral measure → convert to a direct behavioral scale
   - Moving demographics before concept evaluation → refuse and explain why
   - Adding too many open-text questions (>2 per concept) → warn about respondent fatigue

4. **Respect Feedback Scope**:
   - "specific_question": Modify ONLY the targeted question.
   - "section_level": May adjust multiple questions within the section.
   - "add_question": Insert a new question at the most logical position within the cognitive flow. Do not place it randomly.
   - "remove_question": Remove the specified question and re-number remaining ones. Warn if removing an attention check or a core KPI question.

5. **Duration Impact**: If adding questions would push estimated duration beyond 20 minutes, flag this in warnings and suggest which existing questions could be consolidated or removed to compensate.
